# 2026年度入学 履修支援アプリ用データ再構成レポート

## 概要

- 対象: `Course-management-tool` の `autoLoadDepartmentCSVs()` が読む `public/department/{facultyId}/2026/` 配下CSV。
- 旧データは流用せず、アップロードされた2026年度入学向けPDFから、卒業条件・進級/着手条件・教育課程表の科目を再抽出。
- CSVヘッダは既存システム互換: `credit_requirements.csv` は `stage,area,subarea,total_required_credits,必修_credits,選択必修1_credits,選択必修2_credits,自由_credits,notes`、`timetable_by_category.csv` は `id,title,credits,raw_required,category,group,courseType` を含む。
- `○` は `required`、`△/△1/△2` は `elective-required`、無印は `elective` に変換。`※DS/※MS/G/*` は可能な範囲で `raw_required` / `sourceLine` に残した。
- 進級条件と卒業研究/キャップストーン等の着手条件は、各要件CSVの `stage != 卒業` 行と `requirements_2026_summary.json` に格納。現行の集計関数は `stage=卒業` のみ総単位計算するため、画面表示には別セクション実装が必要。

## リポジトリ反映手順

1. ZIP内の `public/department` をリポジトリ直下の `public/department` に上書きする。
2. 開発サーバーで学科選択と入学年度 `2026` を選び、CSV診断で `requirements` と `timetable` が loaded になることを確認する。
3. 進級条件をUIに出す場合は、卒業要件の集計とは分けて `stage=3年次進級 / 4年次進級 / 卒業研究着手` を表示する。
