# 連絡テンプレート集

## 3人に最初に送る共通メッセージ

履修支援アプリの開発を、これからチームで進めたいと考えています。

このアプリは、学生が「卒業要件を満たせるか」「あと何単位必要か」「GPAがどうなるか」「どの科目を取るべきか」を確認できるようにするものです。

最初から難しいことを一気にお願いするのではなく、まずは小さい担当に分けて進めます。

今回の分担は以下の通りです。

- Aさん: 科目データ・卒業要件データの整理
- Bさん: GPA計算・単位集計・時間割重複判定
- Cさん: スマホ向け画面・カードUI作成

開発ルールとして、`main` ブランチには直接pushせず、各自の作業ブランチで作業して、Pull Requestで確認してから統合します。

分からないところは止まって大丈夫です。勝手に判断して進めるより、メモや質問を残してくれる方が助かります。まずは「小さく作って、見せて、直す」形で進めましょう。

## Aさん向け導入メッセージ

Aさんには、Data班として「科目データ」と「卒業要件データ」の整理をお願いしたいです。

プログラミングがまだ不安でも大丈夫です。この担当では、履修要覧や科目表を見ながら、科目名・単位数・必修/選択必修などの区分を整理することが中心です。

最初の目標は、10〜30科目くらいのサンプルデータを作ることです。完璧でなくて大丈夫なので、分からない部分はメモに残してください。

作業ブランチは `data/sample-curriculum` です。

まずは `courses.sample.json` と `requirements.sample.json` を少しずつ作っていきましょう。

## Bさん向け導入メッセージ

Bさんには、Core Logic班として「GPA計算」「単位集計」「時間割重複判定」をお願いしたいです。

画面を作るのではなく、データを受け取って計算結果を返すTypeScript関数を作る担当です。

最初に作るのは、`calculateGpa()`、`summarizeCredits()`、`checkTimetableConflicts()` の3つです。いきなり卒業判定全体を作る必要はありません。

作業ブランチは `feature/core-gpa-and-credit-summary` です。

Reactや画面デザインは触らなくて大丈夫です。まずは小さい関数を1つずつ作り、簡単なテストで確認するところから始めましょう。

## Cさん向け導入メッセージ

Cさんには、Mobile UI班としてスマホ向け画面の作成をお願いしたいです。

このアプリでは、学生がスマホで「卒業できそうか」「あと何単位足りないか」「GPAはどうか」をすぐ見られるようにしたいです。

最初に作るのは、`MobileBottomNav`、`GraduationRiskCard`、`GpaSummaryCard`、`CreditProgressCard`、`CourseCard`、`MobileHomePage` です。

作業ブランチは `feature/mobile-home-ui` です。

最初はダミーデータ表示で大丈夫です。GPA計算や卒業判定のロジックは書かず、まずはスマホで見やすいカードUIを作るところから始めましょう。
