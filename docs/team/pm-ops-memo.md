# 代表者向け運用メモ

## 最初にやること

1. `develop` ブランチを作成する
2. `feature/project-structure` を作成する
3. 基本ディレクトリを作成する
4. Course / Grade / Requirement / Timetable 型を定義する
5. 後輩向けブランチを作成する
6. 各自に説明書を渡す

## 最初のレビュー観点

### Data班

- JSONが壊れていない
- 科目名・単位数・区分が入っている
- 迷いポイントがメモされている
- 不明点を勝手に確定していない

### Core Logic班

- UI依存がない
- 関数が小さい
- 空配列でも落ちない
- テストがある
- 型を勝手に変更していない

### Mobile UI班

- スマホ幅で横スクロールがない
- カードUIである
- ロジックをUIに書いていない
- ダミーデータで表示できる
- ボタン/カードが小さすぎない

## レビュー方針

1. まず動いている点を評価する
2. 良い点を先に伝える
3. 修正点は2〜3個に絞る
4. 一度に大量指摘しない
5. 次に何を直すかを具体的に示す

## 最初の統合目標 (develop)

- サンプル科目データがある
- GPA計算関数がある
- 単位集計関数がある
- スマホホーム画面がある
- ダミーデータで卒業リスク/GPA/不足単位が表示される

## 安全な進行順

1. 代表者が `develop` と作業ブランチを作成
2. 代表者が型定義と docs の土台作成
3. A: サンプルデータ作成
4. B: GPA計算/単位集計/重複判定
5. C: モバイルホームUI
6. develop へ順次マージ
7. ダミーデータでホーム表示確認
